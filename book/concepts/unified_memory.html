Unified Memory Space

Automatic, intelligent, and context-agnostic allocation.

The Concept

In traditional systems programming (like C or Rust), developers must manually decide whether data lives on the Stack or the Heap. Fax-lang introduces Rule 3: Context-Agnostic Allocation, removing this cognitive load while maintaining peak performance.

"The compiler knows better where your data belongs."

Stack

Used for small, short-lived data (usually ≤ 64 bytes) that is frequently accessed. Minimal overhead, maximum speed.

Heap

Used for large objects, dynamic structures, or data that must outlive the current function scope.

Hybrid Allocation

A unique Fax-lang innovation. The compiler can split a single data structure: keeping the "hot" fields on the stack for cache locality and moving "cold" or large fields to the heap transparently.

Why it matters?

  • Reduced Complexity: No more worrying about Box<T> or pointers for simple safety.
  • Deterministic Speed: Optimized for modern CPU cache lines.
  • Memory Safety: Works hand-in-hand with Life-Force to ensure automatic cleanup.
